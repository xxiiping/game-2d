对比对象：

- **本项目**：`scripts/player.gd` + `scenes/player/player.tscn`（自有实现）
- **参考**：`/Users/xp/Godot/Godot-Celeste-Movement-Demo-main-4.x`
  - 控制器：`addons/PixelPerfect2D/Platformer/CharacterController.gd`、`CharacterControllerPlatformer.gd`
  - 数值资源（真正生效）：`addons/PixelPerfect2D/Platformer/ExampleTemplates/Celeste.tres`
  - 玩家场景：`CelesteExample/Scenes/Player.tscn`、关卡：`CelesteExample/Scenes/game.tscn`

> 本文只对比**影响角色位移/手感**的部分，不含头发、残影、粒子、动画、音效等纯表现。本文仅作分析记录与下一步修改参考，**不抄参考源码、不使用其素材**。

## 0. 单位换算说明（重要）

参考控制器把数值资源里的量当作「**每物理帧**」单位，运行时再乘 `physicsfps = 1/delta`。两个项目都用 Godot 默认 **60 物理帧/秒**（均未覆盖 `physics_ticks_per_second`），因此换算固定为：

| 参考量 | 含义 | 换算到 px/s 或 px/s² (60fps) |
|---|---|---|
| 速度类 `X` | 目标速度 = `X * Speed * fps` | `X * 60` px/s |
| 加速度类 `A` | 每帧 `move_toward` 步长 = `A * fps`，每秒应用 60 次 | `A * 60 * 60 = A * 3600` px/s² |
| 时间类 `T` | 秒 | `T` 秒（不变） |

> 注意：参考的「加速度」依赖帧率（`A*fps²`），非帧率无关；我方用 `accel*delta`（帧率无关）。两者在 60fps 下每帧步长完全相同，仅在非 60fps 时才会分叉 —— 当前都跑 60fps，无需处理。

参考空中水平动力学还会再乘 `AirControl`（`Celeste.tres` 中为 **0.5**），即空中所有水平项（加速/刹车/转向）都打 5 折。

## 1. 结论速览（TL;DR）

- **物理常量已约 95% 精确对齐**：最大速度、地面加减速/转向、重力、最大下落、跳跃初速、跳跃截断、土狼时间、跳跃缓冲、墙滑上限、蹬墙跳速度/推力/输入锁/缓冲、墙体检测距离、冲刺速度/上冲缩放/分段锁时长、竖直转角修正、相机硬跟随——**全部一致**（详见对照表）。
- **仍存在 4 项会改变位移轨迹的逻辑差异**（按影响排序）：
  1. **空中转向无专用减速**（最明显）：参考反向时用 990 px/s² 急刹到 0，我方只用 378 px/s² 缓慢拉向反向 → 我方空中掉头明显更“黏”。
  2. **墙滑重力偏强**：参考 288 px/s²，我方 360 px/s²（上限同为 105）。
  3. **冲刺收尾窗口**：参考在 0.075–0.15s 有“锁输入+摩擦归零”过渡，且整段冲刺（0.57s）内**禁用跳跃截断**；我方 0.075s 后立即全控、0.15s 后即恢复跳跃截断。
  4. **水平转角修正触发范围**：参考 ±1..5px 且仅在 `|vy|≈0`；我方 ±1..6px 且 `|vy|≤10.67`，更易触发。
- **2 项 juice 手感参数轻微不同**：命中定格时长（我 50ms / 参考 30ms）、冲刺震屏幅度与方向。
- **存在一处尺度/碰撞差异**（见 §4）：参考玩家碰撞体 **8×8**、世界 320×180@4×、手绘多边形关卡；我方玩家 **12×16**、16px 瓦片、相机 zoom 3。**绝对像素位移已对齐，但相对“角色/瓦片”的观感不同**（同样初速下，参考能跳过约 3.6 个身高，我方约 1.8 个身高）。这是后续要不要进一步对齐的关键决策点。

## 3. 需要修改的差异清单（可直接作为下一步改动）

### 3.1 空中转向缺少专用急刹【优先级：高，最影响手感】

- **参考行为**：空中若输入方向与当前水平速度反向，用 `TurnSpeed*AirControl = 0.55*0.5*3600 = 990 px/s²` 把速度**拉向 0**（先刹停再加速）。
- **我方现状**（`scripts/player.gd` `_process_horizontal`）：空中只要有输入就用 `air_accel = 378` 拉向目标速度，反向时也只有 378 px/s²。
- **效果差异**：从 +90 反向，参考约 5.5 帧（0.09s）刹到 0；我方约 14 帧（0.24s）才到 0 → **我方空中掉头明显迟钝**。
- **改法建议**：在 `_process_horizontal` 的空中分支里，增加“反向判定”：当 `input_x != 0 且 sign(input_x) != sign(velocity.x)` 时，用一个新 `@export var air_turn_decel := 990.0`（拉向 0 或拉向目标均可，拉向 0 更贴参考）。

### 3.2 墙滑重力偏强【优先级：中】

- **参考**：墙滑重力 = `Wall_Gravity * DownGravity * 3600 = 0.1*0.8*3600 = 288 px/s²`。
- **我方**：`wall_slide_gravity = 360`（上限同为 105）。
- **效果差异**：我方更快达到墙滑最大下落速度，贴墙下滑“加速感”略强。
- **改法建议**：`wall_slide_gravity` 由 `360` 改为 `288`。

### 3.3 冲刺收尾与禁跳截断窗口【优先级：中】

- **参考时间轴**（Dash_input_cooldown=0.15）：
  - `0–0.075s`：水平、竖直都锁死（保持冲刺向量）。
  - `0.075–0.15s`：水平解锁但**输入被静默**（目标速度=0，按空中摩擦 720 拉向 0）；竖直仍锁。
  - `0.15s`：恢复正常控制与重力，保留剩余动量。
  - `0–0.57s`（dashIsActive 全程）：**禁用跳跃截断**（冲刺中松跳不削减纵向速度）。
- **我方现状**：`0.075s` 后水平**立即全控**（无摩擦静默过渡）；`0.15s` 后即恢复跳跃截断；另有 0.3s 再触发冷却。
- **效果差异**：我方冲刺出手后能更早重新加速/改向；冲刺上冲后若松跳，我方会在 0.15s 后削减纵向速度，参考则在 0.57s 前不削减。
- **改法建议**（任选其一或组合）：
  - 增加 `dash_input_silence := 0.15`：在 `[ctrl_lock, input_silence]` 区间把水平目标速度置 0、走空中摩擦。
  - 把“禁用跳跃截断”的判定从 `grav_locked` 改为独立的 `dash_active_timer := 0.57`（冲刺视觉/手感窗）。
  - 视需要移除/缩短 `dash_cooldown`（参考无人为冷却，仅靠落地补充次数）。

### 3.4 水平转角修正范围与阈值【优先级：低】

- **参考**：水平修正 `±1..5px`，且仅当 `abs(velocity.y) < 0.22`（≈完全无纵向运动时）。
- **我方**：`±1..6px`，且 `abs(velocity.y) ≤ rise_gravity*delta ≈ 10.67`，更容易触发。
- **改法建议**：`corner_correction_v_px` 由 `6` 改为 `5`；把触发阈值收紧（例如改成接近 0 的小常量，或 `<= max_fall_speed*delta` 的更小值），减少非顶点时刻的“吸附感”。

### 3.5 命中定格时长【优先级：低，juice】

- **参考**：`OS.delay_msec(30)` → 约 **30ms** 硬卡帧（阻塞主线程）。
- **我方**：`time_scale=0` 持续 **50ms**（不阻塞，时间冻结，更干净的实现）。
- **改法建议**：保留 `time_scale` 实现，仅把 `hitstop_time` 由 `0.05` 改为 `0.03` 以对齐时长。

### 3.6 冲刺震屏幅度与方向【优先级：低，juice】

- **参考**：`PixelPerfect2D.shake(dir*1, 0.15)` —— 幅度沿**冲刺方向** ~1px（仅冲刺轴向明显），`ease(time_left/duration, 1.0)` 衰减。
- **我方**：双轴随机 `±1.5px`，线性衰减，0.15s。
- **改法建议**：把 `dash_shake_amp` 由 `1.5` 降到 `1.0`；如需更贴近，可改成沿冲刺方向施加（按 `dir` 分配到 x/y 轴幅度）而非双轴等幅随机。

## 4. 尺度 / 碰撞体差异（决定“相对手感”，需决策）

绝对像素位移已对齐，但**角色与世界的比例不同**，导致相同初速下的“观感”不同：

| 维度 | 参考 | 我方 |
|---|---|---|
| 玩家碰撞体 | **8 × 8** px（脚在原点，y: -8..0） | **12 × 16** px |
| 世界/渲染 | 320×180 视口，整体 4× 放大；关卡是手绘 `CollisionPolygon2D`（**无 TileMap**） | 16px 瓦片网格，相机 `zoom 3` |
| 跳跃绝对高度 | ~29 px | ~29 px（一致） |
| 跳跃相对身高 | ~29 / 8 ≈ **3.6 个身高** | ~29 / 16 ≈ **1.8 个身高** |
| 最大速度相对身宽 | 90 / 8 ≈ **11.3 身宽/秒** | 90 / 12 ≈ **7.5 身宽/秒** |

**含义**：因为我方角色更大、瓦片更大，同样的 90px/s、192px/s 跳跃在“相对世界”里显得更慢、跳得更矮。若你要的“位移效果相同”指**绝对像素**——现已对齐；若指**相对角色/关卡的观感**——需二选一：

- **方案 A（保留现有美术尺度，缩放数值）**：把角色碰撞体改小到接近参考比例（如 8×8 或 8×12），或反过来把所有速度/重力按世界放大倍数整体上调，使“每身高/每瓦片”的位移与参考一致。改动面较大，会牵动关卡与碰撞。
- **方案 B（接受绝对像素对齐）**：维持当前尺度，仅完成 §3 的逻辑/参数对齐。改动小、风险低，绝对轨迹与参考一致，但“相对观感”仍偏“大而慢”。

> 建议：先做 §3（逻辑对齐，收益大、风险低），再根据实机观感决定是否动 §4 的尺度。

## 5. 不影响位移、可忽略

- 地面转向 `move_toward` 拉向 0 vs 拉向目标：60fps 下仅零点跨越那一帧差约 1 帧，忽略。
- 上升/下落重力 640 vs 633.6（差 ~1%）：可顺手把 `rise_gravity`/`fall_gravity` 设为 `634` 精确对齐，亦可不动。
- 冲刺再触发冷却：因 `dash_count=1` 且仅落地补充，实战中与参考无差。
- 输入映射：参考为 D/A/W/S/Space/J；我方额外加了方向键、Z(跳)、Shift(冲)，功能兼容。

## 6. 下一步修改优先级建议

1. **高**：§3.1 空中转向急刹（新增 `air_turn_decel ≈ 990`）。
2. **中**：§3.2 墙滑重力 360→288；§3.3 冲刺禁跳截断窗 + 输入静默过渡。
3. **低**：§3.4 水平转角修正范围/阈值；§3.5 hitstop 0.05→0.03；§3.6 震屏 1.5→1.0/方向化。
4. **决策**：§4 是否对齐尺度（建议先实机体验再定）。
