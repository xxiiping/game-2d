## 项目简介

**Anchor / 回响** 是一款注重手感的硬核 2D 平台跳跃游戏。玩家在像素世界里奔跑、跳跃、蹬墙、冲刺、传送，穿越致命机关，收集散落的星光碎片。

核心设计理念：**每一个机制都有容错** —— 土狼时间给跳崖反悔的窗口，蹬墙缓冲让连跳更跟手，转角修正防止卡棱角。硬核但不恶意，是这款游戏的基本美学。

## 世界观与叙事

> *"锚"是束缚，也是归处。*

一个孤独的角色穿行于破碎的世界。收集星光、触碰锚点、在坠落的边缘反复折返。这不是一个拯救世界的故事 —— 它关于**选择与执念**。

## 核心机制

### 1. 移动系统 —— 数据驱动的物理手感

移动数值与逻辑完全分离，`PlayerMovementConfig` 资源定义了 30+ 个独立调参项：

```
PlayerMovementConfig（Resource）
    ↳ player_default.tres（运行时权威数值）
        ↳ player.gd（纯逻辑，不写死数值）
```

**水平移动**：地面加减速 + 空中加减速 + 方向反转急刹，三套加速度独立调节。

| 参数 | 数值 | 说明 |
|---|---|---|
| `max_speed` | 180 px/s | 最大水平速度 |
| `ground_accel` | 1800 px/s² | 地面同向加速 |
| `ground_friction` | 2448 px/s² | 地面松键摩擦 |
| `air_accel` | 756 px/s² | 空中同向加速 |
| `air_brake` | 1440 px/s² | 空中松键减速 |
| `turn_decel` | 3960 px/s² | 地面反向急刹 |

> 设计要点：空中加速弱于地面，给予跳跃轨迹更强的承诺感；反向急刹最快，让转身反馈清晰。

---

### 2. 跳跃系统 —— 满跳 / 短跳 / 土狼时间 / 输入缓冲

**可变跳跃高度**：松键截断上升速度，实现短跳。

- `jump_velocity = -384 px/s`（起跳初速）
- `jump_cut_factor = 2.0`（松键时速度 ÷ 2，立即转入下落）

**土狼时间（Coyote Time）= 0.1s**  
玩家离开平台后 0.1 秒内仍可起跳。消除"晚了一帧"的挫败感。

**跳跃缓冲（Jump Buffer）= 0.1s**  
在落地前 0.1 秒按跳，落地瞬间自动起跳。消除"早了一帧"的挫败感。

**上升/下落独立重力**：
- `rise_gravity = 1267.2 px/s²`
- `fall_gravity = 1267.2 px/s²`

> 当前上升与下落重力相同，但架构支持独立调节（如需要更飘的顶点或更快的下落）。

---

### 3. 蹬墙跳 —— 墙滑 + 链式连跳 + 墙土狼

贴墙下落时触发墙滑（限速 168 px/s），可蹬墙跳起。

**大跳 / 小跳**：
- 有水平输入：`wall_jump_velocity = -370` + `wall_jump_push = 310`
- 无水平输入：`wall_jump_velocity_min = -250` + `wall_jump_push_min = 210`

**墙土狼（Wall Coyote）= 0.2s**  
离墙后 0.2 秒内仍可蹬墙跳。必须曾经贴过墙，防止凭空蹬墙。

**链式连跳**：下落贴墙时若蹬墙缓冲仍有效，自动刷新缓冲窗口。两面墙之间可无限链式上升。

**蹬墙跳输入锁定 = 0.1s**：起跳后短暂锁定水平输入，保护蹬墙飞出动量不被误操作打断。

---

### 4. 八向冲刺 —— Hitstop + 震屏 + 手感窗

冲刺方向匹配 8 方向摇杆输入，无输入时默认朝面朝方向。

| 参数 | 数值 | 说明 |
|---|---|---|
| `dash_speed` | 456 px/s | 冲刺初速度模长 |
| `dash_up_scale` | 0.75 | 向上冲刺竖直分量缩放 |
| `dash_gravity_lock` | 0.15s | 重力锁定（无重力影响） |
| `dash_control_lock` | 0.075s | 水平控制锁定 |
| `dash_input_silence` | 0.15s | 解锁后忽略输入的静默窗 |
| `dash_active_time` | 0.57s | 手感窗：期内禁止跳跃截断纵向动量 |
| `dash_count` | 1 | 空中可用冲刺次数（落地刷新） |
| `hitstop_time` | 0.03s | 全局定格（time_scale = 0） |
| `dash_shake_amp` | 2.0 px | 相机震动幅度 |

**设计巧思**：
- Hitstop 定格 0.03s 制造"命中感"，虽然只有一瞬，但感知差异巨大
- `dash_active_time` 手感窗保护冲刺纵向动量，松跳键不会意外截断上升
- 重力锁定 + 控制锁定 + 静默窗三阶段分离，精确控制冲刺各阶段手感
- 相机震动沿冲刺方向，幅度随剩余时间衰减

---

### 5. 重坠破瓦 —— 蓄力 → 加速坠落 → 破坏地形

按住 C/R1 蓄力 0.5 秒后进入重坠状态：

- 下落加速 **5x**（`fall_gravity * 5`）
- 接触脆瓦（breakable tile）自动破坏
- 可破坏方块被击中 → 反弹玩家 → 方块销毁
- 按跳跃键或落地退出重坠
- 退出后有 **1s 冷却**

**蓄力取消**：蓄力未满松键即取消，无惩罚。

---

### 6. 传送锚 —— 位置标记与瞬移

短按 Q/L1 → 在当前位置放置锚标记 → 再次短按 → 瞬移回锚点。

长按 0.4 秒 → 取消锚点（播放取消音效）。

- 传送时清零冲刺状态和速度
- 锚标记闪烁提示（附着在玩家身上时 0.3s 间隔闪烁）
- 放置/传送/取消三态独立音效反馈

---

### 7. 传送区域 —— 下落碰撞自动过场

场景中存在隐藏的传送锚区域 `transition_zone` —— 玩家以**重坠状态**落入时触发：

1. 冻结玩家 1.1 秒（播放喵声音效）
2. 自动寻找最近的复活点
3. 瞬移到该位置

这是关卡之间的过渡机制，将叙事与玩法无缝衔接。

---

### 8. 收集品 —— 跟随 → 落地收集 → 结局

玩家触碰收集品后：
- 收集品平滑跟随玩家（lerp 插值）
- 玩家落地时正式收集，计数 +1
- 集齐全部 → 自动触发结局场景
- 死亡时收集品回到原位（不丢失进度）

UI 实时显示收集进度：★ 3 / 7

---

### 9. 存档系统 —— 自动存档 + 最近复活点

每 5 秒自动存档到 `user://save.json`：
- 保存最近的复活点位置（只在玩家身后的 checkpoint）
- 记录收集品进度
- 主菜单可继续游戏 / 新游戏

---

### 10. 转角修正 —— 不卡棱角

玩家上升顶头被挡时，左右微移尝试滑过棱角（`corner_correction_px = 8px`）。  
水平移动被挡时，上下微移尝试滑过（`corner_correction_v_px = 10px`）。

这是平台游戏手感的"隐秘工程"—— 玩家不会注意到它，但没有它一定会抱怨。

---

### 11. 可破坏方块 —— 冲刺/重坠撞击

- 检测区域比碰撞形状大 12px，确保高速度下不会穿透
- 被击中时反弹玩家（沿远离方向 200 px/s）
- 播放破坏动画和音效，0.5s 后销毁
- 冲刺和重坠均可触发

---

### 12. 死亡与重生

- 落入 kill_zone → 播放死亡动画 → 自动回到最后一个复活点
- 重坠状态中落入 kill_zone **不触发死亡**（保护重坠玩法）
- 死亡动画期间冻结所有输入
- R 键/右摇杆按下 → 手动重新载入场景

## 系统架构

```
project.godot             项目配置：视口/输入映射/物理引擎/自动加载
    │
    ├── Global (Autoload)       跨场景全局状态（出生位置）
    ├── SaveManager (Autoload)  存档管理 + 收集品计数 + 自动保存
    │
    ├── scenes/
    │   ├── player/             玩家场景（CharacterBody2D + 动画 + 音效）
    │   ├── levels/             关卡（TileMap + 装饰 + 对象摆放）
    │   ├── objects/            交互对象（checkpoint / kill_zone / transition / breakable）
    │   └── ui/                 菜单 / 结局 / HUD
    │
    ├── scripts/
    │   ├── player.gd           玩家核心逻辑（~600 行）
    │   └── resources/
    │       └── player_movement_config.gd   移动数值 Resource 类定义
    │
    └── resources/
        └── movement/
            └── player_default.tres         当前正式数值（调手感只改这里）
```

### 数据流设计

```
player_movement_config.gd    定义字段 + 出厂默认 + 编辑器文档注释
         ↓
player_default.tres          运行时权威数值（在 Godot 检查器中调参）
         ↓
player.tscn → player.gd      读取 movement.xxx，纯逻辑，不写死数字
```

这是平台游戏手感调参的最佳实践：**改数值不改代码**。设计师（或未来的自己）只需打开 `.tres` 文件拖动滑块，无需触碰一行代码。

## 输入设计

### 双模式输入

| 动作 | 键盘 | PS5 手柄 |
|---|---|---|
| 移动 | WASD | 左摇杆 / 十字键 |
| 跳跃 | Space | × (Cross) / △ (Triangle) |
| 冲刺 | Shift | ○ (Circle) / □ (Square) |
| 重坠 | C | R1 |
| 放置/传送锚 | Q | L1 |
| 重新开始 | R | R3 |

- 键盘与手柄**同时可用**，无缝切换
- 死区统一 **0.2**（移动）/ **0.5**（UI 导航）
- 跳跃与冲刺各绑定两个手柄按键，适配不同握持习惯
- Android 平台禁用触屏，仅手柄操作

## AI 协作开发

本项目是人机协作的游戏开发实践。AI（Claude，Anthropic）参与的工作包括但不限于：

| 领域 | AI 贡献 |
|---|---|
| **移动系统** | 土狼时间、跳跃缓冲、蹬墙跳链式连跳的完整实现 |
| **冲刺系统** | 八向冲刺 + hitstop + 相机震动 + 三阶段锁定分离 |
| **重坠破瓦** | 蓄力状态机 + 脆瓦网格破坏 + 冷却系统 |
| **传送锚** | 放置/传送/长按取消三态交互 |
| **转角修正** | 上下左右双向棱角滑过算法 |
| **代码架构** | PlayerMovementConfig 数据驱动设计、存档系统 |
| **输入映射** | 键盘+手柄双模式配置 |
| **导出配置** | PCK 加密、多平台导出预设、Android 签名 |
| **版本控制** | `.gitignore` 规则设计、敏感文件排除 |

**开发模式**：人类负责创意方向、玩法设计决策、手感调参；AI 负责代码实现、架构建议、问题排查。通过 Godot MCP（Model Context Protocol）工具，AI 可以直接与运行中的 Godot 编辑器交互 —— 读取场景、修改脚本、运行测试 —— 形成"说 → 改 → 跑 → 调"的实时循环。
